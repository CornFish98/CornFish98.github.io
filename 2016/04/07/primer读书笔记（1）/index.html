primer读书笔记（1）

1. 变量和基本类型

1.1 基本类型

C++的内置类型包括算数类型和空类型，其中算数类型分为整型和浮点型。C++标准规定了每种类型的最小尺寸，但同时允许编译器赋予这些类型更大的尺寸。

一个char的大小和一个机器字节一样（8位），wchar_t可存放机器最大拓展字符集中的任意字符，char16_t和char32_t在Unicode字符集中使用。字符型被分为三种，char、sighed char、unsigned char。字符只分为带符号和无符号的，至于char类型到底带不带符号与编译器有关。

一个int（最少16位）至少和一个short一样大，一个long至少和一个int一样大，一个long long至少和一个long一样大，这是C++11中新定义的类型。（对于大部分编译器，long通常和int一样大，因此当数值超过int时，采用long long）

对于浮点型，由于大部分机器1个字（32位）为4个字节，因此通常float以1个字表示，double以2个字表示。（float和double分别有7和16个有效位）

当赋给带符号类型一个超出它表示范围的数时，结果是未定义的。编程时，不应假设某种类型的大小（同一种类型在不同机器可能有不同的大小），否则写出的程序是不可移植的（nonportable）。

每个字面值常量都对应一种数据类型。short没有对应的字面值，十进制字面值的类型是int、long、long long中能容纳它的最小者，八进制或十六进制的字面值类型是int、unsigned int、long、unsigned long、long long和unsigned long long中能容纳它的最小者。

注意区分字符字面值和字符串字面值，字符串字面值的类型实际上是由常量字符构成的数组，并且编译器会在它末尾添加一个‘\0’。

C++规定了以反斜线作为开始的转义序列（如\n、\），此外还可以使用泛化的转义序列，其形式是\x紧跟一个或多个十六进制数，或\后紧跟1-3个八进制数（超过3个则只有前3个与\构成转义序列），为了防止\x开头的转义序列越界（即超过char型的最大表示范围），通常在它前面加前缀（u、U、L、u8等）一起使用。

1.2 变量

注意区分初始化和赋值的区别，初始化是在创建变量时赋予其一个初始值，而赋值的含义则是把当前值擦除，并以一个新值来代替。在C++11中，用花括号初始化变量得到了全面的应用，因此对一个变量进行初始化时，可以使用=进行初始化，也可以直接使用初始化列表，但使用初始化列表时必须赋予合适类型的值，初始化列表不支持隐式转换（例如不能在初始化列表中将一个double值赋给一个int型变量）。

如果定义变量时没有赋初值，此时，对于内置类型，如果变量定义于函数体外，则被默认初始化为0，如果该内置类型定义于函数体内，则将不被默认初始化（即其值时未定义的）。对于类类型，每个类各自决定其初始化对象的方式，是否允许不经初始化就定义对象也由类自己决定（如果类允许这种行为，它将决定对象的初值到底是什么）。

为了支持分离式编译，C++对变量的声明和定义进行了区分，声明使得名字为程序所知（一个文件如果想使用别处定义的变量则必须包含对该变量的声明），而定义负责创建与名字相关的实体（申请存储空间，并可能赋一个初始值）。在变量名前加上extern关键字来声明一个变量（注意，如果声明的同时赋予了初始值，此时就不再是声明，而属于定义了，也就是说extern关键字被忽略了）。变量可以被声明多次，但只能被定义一次。

变量的命名规范和作用域次数略去（注意掌握作用域运算符”::”）.

1.3 复合类型

复合类型包括引用和指针。

引用在声明的同时也必须被初始化，并且一旦初始化后，它就和它的初始值绑定在一起并且再无法改变。引用即别名，它并非对象，它只是为一个已经存在的对象所起的另外一个名字。由于引用不是对象，因此无法定义引用的引用。

指针本身是一个对象，允许对指针拷贝和赋值，一个指针在它的声明周期内可以先后指向不同的对象。指针无需在定义时赋初值，和其他内置类型一样，如果在块作用域内定义的指针没有被初始化，它将拥有一个不确定的值（未定义的值）。由于指针是对象，因此存在指针的指针和指针的引用。

可以用字面值nullptr初始化指针来得到空指针。nullptr是C++11中新定义的一种特殊类型的字面值，它可以被转化成任意其他指针类型。在旧标准中，通常使用预处理变量NULL（定义在头文件cstdlib中）生成空指针。

void 指针是一种特殊的指针类型，可用于存放任意类型对象的地址，它只能用于和其它指针比较或赋值给另外一个指针，不能通过解引用运算符 来访问它所指向的对象（因为并不知道类型）。

注意指针的定义：int** p1, *p2, p3;//这行代码中，p1是int型指针的指针，p2是int型指针，p3是int型的数。

1.4 const和constexpr

在变量类型前加上const关键字表示定义了一个不能被修改的常量，const对象在定义时必须被初始化，注意这里初始值可以是任意复杂的表达式。我们可以用一个非常量初始化一个const常量，这是因为常量特征仅在执行改变它的操作发生时才会发生作用，这里再次说明了初始化不同于赋值（赋值会改变原先的值）。

常量引用是对常量的引用（对const的引用），通常引用的类型必须与其所引用的对象类型一致，但在初始化常量引用时允许用任意表达式作为初始值，只要该表达式的结果可以转换为引用的类型即可（允许为一个常量引用绑定非常量对象、字面值，甚至一个表达式）。注意，对const的引用可能引用一个并非const的对象，只是我们不能通过对const的引用修改该对象的值。

注意区分指向常量的指针、常量指针和指向常量的常量指针。

指向常量的指针强调的是：这个指针不能用于改变其所指对象的值。要想存放常量对象的地址，必须使用指向常量的指针。通常，指针的类型必须与其所指对象的类型一致，但是，指向常量的指针是个例外，C++并不规定指向常量的指针所指的对象必须是一个常量，所谓指向常量的指针仅仅要求不能通过该指针改变对象的值（如果这个对象不是常量，是可以通过其他途径改变的）。

这里有必要再补充一点，很多人对上面的定义存在困惑，认为指向常量的指针指向的就应该是常量，产生这样的观点其实并不奇怪，事实上，如果我们站在指向常量的指针的角度看这个的话，答案确实如此，也就是说，指向常量的指针自己认为自己指向的对象是一个常量，因此它“自觉地”不去修改它所指向的这个对象。但如果站在旁观者的角度，这个对象（如果确实不是常量的话）是可以通过其他方式修改的。这样的做法并不奇怪，而且是有用意的，而且我们也遇到过类似的用法（考虑我们定义一个int型常量const int val，在对val初始化时，并非必须使用常量才能初始化它，我们也可以使用普通的int型变量初始化它）。

之所以要讨论顶层const和底层const的区别，是因为在执行对象的拷贝操作时，顶层const不会产生影响，但底层const会。只有当拷入和拷出的对象具有相同的底层const时（或两个对象的数据类型必须能够转换），才能执行拷贝操作。

1.6 auto与decltype

C++11引入auto类型说明符，用它能够让编译器替我们去分析表达式所属的类型，auto定义的变量必须有初始值。

首先，由于使用引用其实使用的是引用的对象，因此当auto一个引用时，编译器将以引用对象的类型作为auto的类型。如：int i = 0， &r= i; auto a = r;此时编译器会推断a的类型为int，而非一个int型的引用（关于这一点，decltype恰好相反，它会将类型推断为引用）
其次，auto一般会忽略顶层const，保留底层const，如果希望推断的auto类型是一个顶层const，需要明确指出，如：const int ci = 0; const auto f = ci; 此时ci的推演类型是int，f的推演类型是const int。

C++11中还引入了第二种类型说明符decltype，其作用是选择并返回操作数的数据类型，在此过程中，编译器分析表达式并得到它的类型，但是却不计算表达式的值。decltype处理顶层const和引用的方式与auto不同，如果decltype使用的表达式是一个变量，则decltype返回该变量的类型（包括顶层const和引用在内），下面的例子说明了这一点：
